# AWS S3 Image Resize Lambda with Temporary EOD Bucket Cleanup

## Project Overview

This project implements a secure AWS serverless workflow that:

* Automatically resizes images uploaded to an S3 bucket.
* Stores resized images in a second S3 bucket.
* Uses an EventBridge scheduled job to empty the resized bucket at End of Day (EOD).
* Removes the scheduled cleanup job after verification to prevent repeated deletions, accidental data loss, or unnecessary charges.

Technologies used:

* Amazon S3
* AWS Lambda
* AWS IAM
* Amazon EventBridge Scheduler
* Amazon CloudWatch Logs
* Python 3.12
* Pillow (PIL)

## Architecture Workflow

```text
Upload image to source bucket
        ↓
S3 trigger invokes resize-image-lambda
        ↓
Lambda resizes image using Pillow
        ↓
Resized image stored in destination bucket
        ↓
EventBridge Scheduler invokes cleanup Lambda at EOD
        ↓
Destination bucket emptied
        ↓
Scheduler removed after verification
```

---

## Step 1 – Create S3 Buckets

Two S3 buckets were created:

### Source Bucket

`original-images-task1-2026`

Used for storing original uploaded images.

### Destination Bucket

`resized-images-task1-2026`

Used for storing resized output images.

---

## Step 2 – Create IAM Policies

Least privilege access was applied.

### Resize Policy

**Policy Name:** `LambdaImageResizePolicyTask1`

Permissions granted:

* Read objects from:

```text
original-images-task1-2026/originals/*
```

* Write objects to:

```text
resized-images-task1-2026/*
```

* Write CloudWatch logs

### Cleanup Policy

**Policy Name:** `LambdaCleanupBucketPolicy`

Permissions granted:

* List destination bucket
* Delete objects inside:

```text
resized-images-task1-2026/*
```

* Write CloudWatch logs

This policy does not allow access to the original bucket.

---

## Step 3 – Create IAM Roles

### Resize Lambda Role

`resize-lambda-role`

Attached policy:

`LambdaImageResizePolicyTask1`

### Cleanup Lambda Role

`cleanup-lambda-role`

Attached policy:

`LambdaCleanupBucketPolicy`

### Scheduler Role

Automatically generated by AWS:

`Amazon_EventBridge_Scheduler_LAMBDA_9e5e32fbba`

Used by EventBridge Scheduler to invoke Lambda.

---

## Step 4 – Create Resize Lambda Function

### Function Name

`resize-image-lambda`

### Runtime

`Python 3.12`

### Purpose

When an image is uploaded to:

```text
original-images-task1-2026/originals/
```

Lambda automatically:

1. Downloads the image
2. Resizes the image
3. Uploads resized output to:

```text
resized-images-task1-2026
```

### Dependency Used

`Pillow (PIL)`

---

## Step 5 – Configure S3 Trigger

The source bucket was connected to Lambda using an `ObjectCreated` trigger.

Prefix used:

```text
originals/
```

Whenever a new image is uploaded, Lambda runs automatically.

---

## Step 6 – Create Cleanup Lambda Function

### Function Name

`empty-resized-bucket-lambda`

### Purpose

Deletes all objects inside:

```text
resized-images-task1-2026
```

Used for scheduled EOD cleanup.

---

## Step 7 – Configure CloudWatch Logs

Lambda execution logs were stored in:

```text
/aws/lambda/resize-image-lambda
```

Used for testing, debugging, and verification.

---

## Step 8 – Create EventBridge Scheduler

### Schedule Name

`temporary-empty-resized-bucket-eod`

### Target Lambda

`empty-resized-bucket-lambda`

### Timezone Used

`Asia/Karachi`

### Purpose

Invoke cleanup Lambda at End of Day.

The scheduler was configured as temporary in order to meet the requirement of removing it after verification.

## Security Best Practices Applied

### Least Privilege IAM

Each Lambda function only received permissions necessary for its task.

### Separation of Storage

Original and resized images were stored in separate buckets.

### Temporary Automation

Cleanup schedule was not left permanently active.
